# DoubleUI Transaction Entry Automation

## Overview
This UiPath automation processes a batch of deposit transactions using UiPath's DoubleUI training application — it reads transaction data from Excel, enters each one into the app, captures the generated transaction number, and writes it back to the spreadsheet.

## What it does
1. Opens the DoubleUI desktop application
2. Reads transaction records (Cash In and On Us Check amounts) from `DoubleUI-Transactions.xlsx` into a DataTable
3. Loops through each row in the DataTable:
   - Types the Cash In amount into the app
   - Types the On Us Check amount into the app
   - Clicks "Accept" to submit the transaction
   - Reads the Transaction Number generated by the app
   - Writes that transaction number back into the corresponding row in the Excel file
4. Repeats until all transactions in the spreadsheet have been processed

## Workflow
- **Main.xaml** — contains the full automation logic (open app → read Excel → loop through rows → enter data → capture transaction number → update Excel)

## Output
- `DoubleUI-Transactions.xlsx`, updated with the generated Transaction Number for each row
- This Excel file is included in the folder, already populated with the output from a completed run

## Key concepts practiced
- Desktop application automation (non-browser UI Automation)
- Excel read/write operations
- Looping through DataTable rows (For Each Row)
- Reading dynamically generated values from an application and feeding them back into a spreadsheet

## Requirements
- UiPath Studio
- UiPath.Excel.Activities
- UiPath.UIAutomation.Activities
